# 使用 Create LIFF App 建構 LIFF app 開發環境

## What is Create LIFF App 

Create LIFF App 是一款 CLI 工具，讓你只用一道指令就能建構 LIFF app 的開發環境，就像 React 的 [Create React App](https://github.com/facebook/create-react-app) 或 Next.js 的 [Create Next App](https://nextjs.org/docs/pages/api-reference/cli/create-next-app) 一樣。回答 Create LIFF App 提出的問題後，便會產生包含 LIFF app 範本在內的環境，讓你可以立即開始開發。

- [GitHub](https://github.com/line/create-liff-app)
- [npm](https://www.npmjs.com/package/@line/create-liff-app)

如需進一步瞭解 LIFF starter app、LIFF Playground 與 Create LIFF App 之間的差異，請參閱「[支援 LIFF app 開發的工具](https://developers.line.biz/en/docs/liff/overview/#support-tool)」。

## Languages and frameworks 

Create LIFF App 目前可使用以下語言與框架。例如，你可以產生以 TypeScript 撰寫的 Next.js 原始碼。

### Languages that can be generated by the Create LIFF App 

- JavaScript
- TypeScript

### Frameworks that can be generated with Create LIFF App 

- [Next.js](https://nextjs.org/)
- [Nuxt](https://nuxt.com/)
- [React](https://react.dev/)
- [Vue.js](https://vuejs.org/)
- [Svelte](https://svelte.dev/)

## Operating environment 

Create LIFF App 在 Node.js 上執行。套件管理你可以使用 Yarn 或 npm，但本頁的說明使用 Yarn。本頁內容已在以下各版本上測試過：

| 名稱                                                       | 版本    |
| ---------------------------------------------------------- | ------- |
| [Create LIFF App](https://github.com/line/create-liff-app) | 1.1.0   |
| [Node.js](https://nodejs.org/en)                           | 18.17.1 |
| [Yarn](https://yarnpkg.com/)                               | 1.22.19 |

## Advance preparation 

執行 Create LIFF App 時需要 LIFF ID。請先閱讀「[建立頻道](https://developers.line.biz/en/docs/liff/getting-started/)」與「[將 LIFF app 新增至頻道](https://developers.line.biz/en/docs/liff/registering-liff-apps/)」，以取得你的 LIFF ID。

<!-- tip start -->

**輸入端點（endpoint）URL**

執行 Create LIFF App 時，你需要輸入 **Endpoint URL**。由於端點 URL 之後可以修改，因此此時你可以先輸入一個暫時性的 URL（例如 `https://example.com/`）。

<!-- tip end -->

## Using the Create LIFF App 

在接下來的兩個步驟中，請使用 Create LIFF App 建立 LIFF app 的開發環境，並在本機確認其可正常運作。

1. [使用 Create LIFF App 建立開發環境](https://developers.line.biz/en/docs/liff/cli-tool-create-liff-app/#create-a-dev-env-using-liff-app)
1. [在 localhost 上啟動 LIFF app](https://developers.line.biz/en/docs/liff/cli-tool-create-liff-app/#start-the-liff-app-on-the-localhost)

### Create a development environment using the Create LIFF App 

1. 執行 Create LIFF App。開啟終端機或命令列工具（以下簡稱「終端機」）。在任意目錄下執行以下指令：

   ```bash
   $ npx @line/create-liff-app
   ```

   請注意，執行指令時可以指定[選項](https://developers.line.biz/en/docs/liff/cli-tool-create-liff-app/#options)。

1. 接下來你將回答 Create LIFF App 提出的問題。如果想在回答問題的過程中中止，可以在 Windows 上按 Ctrl+c、在 macOS 上按 control+c 來中斷。

1. 輸入專案名稱。如果不輸入任何內容，專案名稱將預設為 `my-app`。專案名稱會用作 Create LIFF App 所產生的目錄名稱，以及其他用途。

   ```bash
   ? Enter your project name:  (my-app)
   ```

   以下步驟假設你在專案名稱中輸入了 `my-app`。

1. 選擇你想使用的程式庫或框架。

   ```bash
   ? Which template do you want to use? (Use arrow keys)
   ❯ vanilla
     react
     vue
     svelte
     nextjs
     nuxtjs
   ```

1. 選擇你想使用的語言。

   ```bash
   ? JavaScript or TypeScript? (Use arrow keys)
   ❯ JavaScript
     TypeScript
   ```

1. 輸入 LIFF ID。你也可以不輸入 LIFF ID 直接繼續。如果想稍後再輸入或變更 LIFF ID，請編輯所產生的 `my-app` 目錄中的 `.env` 檔案。

   ```bash
   ? Please enter your LIFF ID:
   Don't you have LIFF ID? Check out https://developers.line.biz/ja/docs/liff/getting-started/ (liffId)
   ```

1. 選擇要使用哪一個套件管理工具。套件管理你可以使用 Yarn 或 npm，但此處我們使用 Yarn。

   ```bash
   ? Which package manager do you want to use? (Use arrow keys)
   ❯ yarn
   npm
   ```

1. 當你回答完所有問題後，將會在 `my-app` 目錄下產生包含 LIFF app 範本的開發環境。

   ```bash
   yarn install v1.22.19
   warning package.json: No license field
   info No lockfile found.
   warning my-app@0.0.0: No license field
   [1/4] 🔍  Resolving packages...
   [2/4] 🚚  Fetching packages...
   [3/4] 🔗  Linking dependencies...
   [4/4] 🔨  Building fresh packages...
   success Saved lockfile.
   ✨  Done in 25.06s.

   Done! Now run:

     cd my-app
     yarn dev
   ```

   #### Options {#options}

   Create LIFF App 接受下表所列的選項。例如，要產生以 TypeScript 撰寫的 Next.js 原始碼，請執行以下指令。請注意，在選項中指定的項目將會從 Create LIFF App 的問題中省略。

   ```bash
   $ npx @line/create-liff-app -t nextjs --ts
   ```

   | 簡短選項 | 完整選項 | 引數 | 行為 |
   | --- | --- | --- | --- |
   | -v | --version |  | 顯示版本號 |
   | -t | --template | &lt;template&gt; | 指定範本<br>引數可選值：`vanilla`、`react`、`vue`、`svelte`、`nextjs`、`nuxtjs` |
   | -l | --liffid | &lt;liff id&gt; | 指定 LIFF ID |
   | --js | --javascript |  | 產生以 JavaScript 撰寫的原始碼 |
   | --ts | --typescript |  | 產生以 TypeScript 撰寫的原始碼 |
   | --npm | --use-npm |  | 使用 npm 作為套件管理工具 |
   | --yarn | --use-yarn |  | 使用 Yarn 作為套件管理工具 |
   | -h | --help |  | 顯示指令的說明 |

### Start the LIFF app on the localhost 

1. 讓我們在 localhost 上啟動所產生的 LIFF app。要啟動 LIFF app，請執行 `yarn dev` 指令。當出現以下訊息且終端機畫面輸出停止時，表示 LIFF app 正在本機伺服器上執行。

   ```bash
   $ yarn dev
   yarn run v1.22.19
   warning package.json: No license field
   $ vite

     vite v2.9.13 dev server running at:

     > Local: http://localhost:3000/
     > Network: use `--host` to expose

     ready in 170ms.
   ```

1. 當你用瀏覽器存取終端機中顯示的 URL（vanilla JavaScript 為 `http://localhost:3000`）時，你會看到顯示 `LIFF init succeeded.` 訊息的畫面，如下所示。

   ![Success](https://developers.line.biz/media/liff/cli-tool-create-liff-app/create-liff-app-success.png)

   如果你尚未設定 LIFF ID，你會看到顯示 `LIFF init failed.` 訊息的畫面，如下所示。請將 LIFF ID 寫入所產生的 `my-app` 目錄中的 `.env` 檔案，並重新啟動本機伺服器。

   ![Failure](https://developers.line.biz/media/liff/cli-tool-create-liff-app/create-liff-app-failed.png)

1. 在瀏覽器中確認 LIFF app 正常執行後，請在 Windows 上按 Ctrl+c、在 macOS 上按 control+c 來停止本機伺服器。

## Next step 

現在你已準備好開發你的 LIFF app。

若要在 LINE 上確認 LIFF app 的運作，請將所產生的 LIFF app 部署到任意伺服器（例如 Netlify），並將該 URL 設定為 **Endpoint URL**。本頁不說明如何部署到 Netlify，也不說明如何在 LINE 上確認 LIFF app 的運作。請參閱「[將 LIFF starter app 部署到伺服器](https://developers.line.biz/en/docs/liff/trying-liff-app/#deploy-to-server)」。

如需進一步瞭解實際的開發內容，請參閱「[開發 LIFF app](https://developers.line.biz/en/docs/liff/developing-liff-apps/)」。
